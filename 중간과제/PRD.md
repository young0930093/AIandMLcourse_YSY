# PRD (Product Requirements Document)

## 한끼 칼로리 트래커 (Meal Calorie Tracker)

**작성자:** 영이
**과목:** AI와 머신러닝 (부산대학교 물리학과)
**작성일:** 2026년 4월 21일
**프로젝트 기간:** 2026년 4월 22일 ~ 4월 29일

---

## 1. 프로젝트 개요

### 1.1 한 줄 소개
사용자가 반찬 사진을 하나씩 업로드하면 CNN이 음식을 자동 인식해 칼로리와 탄수화물·단백질·지방(탄단지) 성분을 분석하고, 총 섭취 칼로리를 운동량으로 환산해 일일 영양 목표 달성률을 보여주는 웹 애플리케이션.

### 1.2 배경 및 문제 정의
기존 칼로리 추적 앱은 사용자가 음식 이름을 직접 검색하고 입력해야 하는 번거로움이 있다. 또한 단순 칼로리만 보여주는 경우가 많아 다이어트나 근력 운동을 위한 영양 관리(탄단지 균형)에 한계가 있다. 본 프로젝트는 CNN 기반 이미지 분류로 입력 과정을 간소화하고, 칼로리뿐 아니라 탄수화물·단백질·지방 비율까지 자동 계산해 사용자가 영양 균형을 직관적으로 파악할 수 있도록 하는 것을 목표로 한다.

### 1.3 목적
- 수업에서 학습한 머신러닝 기술(선형 회귀, MLP, CNN, Transfer Learning)을 실제 웹 애플리케이션에 통합하는 경험
- 이미지 분류 모델을 실사용 환경에 배포하는 전체 파이프라인 구축
- 칼로리 + 탄단지 영양 성분 분석을 통한 실용적 가치 창출

### 3.2 사용자 플로우 상세

**[Step 0] 첫 접속 시 - 프로필 생성**
- 처음 사이트 접속 시 프로필이 없으면 자동으로 생성 화면 표시
- 입력 항목:
  - 이름 (별명 가능)
  - 성별: 남 / 여
  - 나이
  - 키 (cm)
  - 몸무게 (kg)
  - 활동량: 적음 / 보통 / 많음
- [생성] 버튼 → Harris-Benedict 공식으로 개인 맞춤 권장 섭취량 자동 계산
  - 기초대사량(BMR)
  - 일일 총 에너지 필요량(TDEE)
  - 탄수화물/단백질/지방 일일 목표량
- 생성 완료 후 홈 화면으로 이동

**[Step 0-1] 재접속 시 - 프로필 선택**
- 이미 생성된 프로필이 있으면 프로필 선택 화면 표시
- 여러 프로필이 있을 경우 모두 나열 (예: "영이", "언니")
- [+ 새 프로필] 버튼으로 추가 생성 가능
- 프로필 선택 시 해당 프로필의 목표량과 히스토리가 로드됨
- 프로필 수정/삭제 가능

**[Step 1] 홈 화면**
- 현재 프로필 이름 표시 + 프로필 전환 버튼
- "오늘 뭐 먹었어?" 메인 메시지와 함께 사진 업로드 버튼 제공
- 오늘 누적 섭취 미리보기 (칼로리 + 탄단지 목표 대비 %)
- 최근 식사 기록 미리보기 (날짜, 총 칼로리)
- 히스토리 페이지로 이동 버튼

**[Step 2] 사진 업로드 화면**
- 반찬 하나씩 사진 추가 가능 ([+ 사진 추가] 버튼)
- 각 사진 업로드 시 CNN이 즉시 음식 인식 시도
- 인식 결과 표시: 음식명 + 신뢰도 + 예상 칼로리

**[Step 3-A] 인식 성공 시**
- 결과 예시: "🍲 김치찌개 (신뢰도 87%) / 150kcal"
- 사용자 선택: [✓ 맞아] 또는 [✗ 다른 음식이야]
- "✗ 다른 음식이야" 선택 시 → 힌트 모드로 전환

**[Step 3-B] 인식 실패 시 (신뢰도 낮음)**
- 결과 예시: "??? (신뢰도 낮음)"
- 사용자 선택: [💡 힌트 주기] 또는 [직접 입력]

**[Step 4] 힌트 모드**
- 음식 카테고리 선택지 제공:
  - 🍚 밥/면류
  - 🍲 국/찌개
  - 🥬 나물/채소
  - 🍖 육류
  - 🐟 해산물
  - 🍳 계란류
- 카테고리 선택 시 해당 카테고리 내에서 CNN 재분류
- 좁은 카테고리 내 분류이므로 정확도 대폭 향상

**[Step 5] 결과 화면**
- 섭취한 음식 리스트와 각 칼로리 + 탄단지(g) 표시
- 총 칼로리 하이라이트
- 한끼 총 영양 성분 섹션:
  - 🌾 탄수화물: X g
  - 🍖 단백질: X g
  - 🧈 지방: X g
- 오늘 누적 영양 성분 진행 바 (목표 대비 %):
  - 탄수화물 ████████░░ 60% (목표 300g)
  - 단백질   ██████░░░░ 55% (목표 90g)
  - 지방     ██████░░░░ 61% (목표 65g)
- 운동량 환산 섹션:
  - 🏃 달리기 약 X km
  - 🚶 걷기 약 X 분
  - 🚴 자전거 약 X 분
- [저장] 버튼으로 히스토리에 기록

**[Step 6] 히스토리 화면 (P1)**
- 일별 섭취 칼로리 막대 그래프
- 주간/월간 추세선 (선형 회귀 기반)
- 평균 일일 칼로리 표시
- 일별 탄단지 비율 도넛 차트
- 주간 평균 탄단지 섭취량 추이

## 5. 데이터 요구사항

### 5.1 학습 데이터
- Food-101, Kaggle Food Images 등 공개 데이터셋 활용
- 필요 시 직접 수집한 보완 데이터 추가

### 5.2 영양 정보 DB
- 음식별 칼로리 + 탄수화물(g) + 단백질(g) + 지방(g) 정보 DB 구축 (JSON)
- 각 음식을 소/중/대 사이즈별로 영양 정보 저장
- 참고 출처: 식약처 식품영양성분 DB, USDA FoodData Central
- 백엔드에 정적 파일로 포함 (변경 없음)

### 5.3 개인 맞춤 권장 섭취량 계산

**기초대사량(BMR) - Harris-Benedict 공식:**
- 남: `88.362 + (13.397 × 몸무게kg) + (4.799 × 키cm) - (5.677 × 나이)`
- 여: `447.593 + (9.247 × 몸무게kg) + (3.098 × 키cm) - (4.330 × 나이)`

**일일 총 에너지 필요량(TDEE):**
- TDEE = BMR × 활동계수
  - 적음: 1.2 (거의 운동 안 함)
  - 보통: 1.55 (주 3~5회 운동)
  - 많음: 1.725 (주 6~7회 강도 높은 운동)

**탄단지 일일 목표량:**
- 탄수화물: TDEE의 55% ÷ 4 (g)
- 단백질: 몸무게(kg) × 1.0 (g)
- 지방: TDEE의 25% ÷ 9 (g)

**계산 예시:**
- 여성, 25세, 165cm, 55kg, 보통 활동량
- BMR = 447.593 + (9.247×55) + (3.098×165) - (4.330×25) ≈ 1358 kcal
- TDEE = 1358 × 1.55 ≈ 2105 kcal
- 탄수화물 목표: (2105 × 0.55) / 4 ≈ 289 g
- 단백질 목표: 55 × 1.0 = 55 g
- 지방 목표: (2105 × 0.25) / 9 ≈ 58 g

### 5.4 데이터 저장 전략

**저장 방식:** 브라우저 localStorage 기반 (서버 DB 불필요)

**저장 이유:**
- 로그인/회원가입 없이 즉시 사용 가능
- 배포 서버 이슈와 무관하게 데이터 유지
- 브라우저를 닫거나 컴퓨터를 재부팅해도 기록 보존
- 과제 제출 및 시연에 가장 안정적인 방식

**제약 사항:**
- 같은 브라우저에서만 기록 유지 (다른 기기/브라우저에선 별도 관리)
- 브라우저 데이터 삭제 시 기록 소실 → 백업/복원 기능으로 보완

**다중 프로필 지원:**
- 한 브라우저에 여러 프로필 저장 가능
- 프로필 전환 시 해당 프로필의 데이터만 표시
- 가족/친구와 같은 기기를 공유해도 각자 데이터 관리 가능

### 5.5 운동량 환산 공식
- 개인 체중 기반 칼로리 소모량 계산
  - 달리기: 약 `(체중kg × 1.036)` kcal/km
  - 걷기: 약 `(체중kg × 0.067)` kcal/min
  - 자전거: 약 `(체중kg × 0.133)` kcal/min
- Week 1-2 선형 회귀로 사용자별 운동 기록 기반 보정 가능 (향후 확장)
